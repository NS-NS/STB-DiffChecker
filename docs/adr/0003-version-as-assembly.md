# 0003: ST-Bridgeバージョン=アセンブリ + partial classでの比較定義

- ステータス: 採用（2026-07。元設計の「バージョン=exe」をライブラリに再編）

## 文脈
ST-Bridgeのバージョン(2.0.1/2.0.2/2.1.0)ごとにスキーマが異なり、
XSDから生成したクラス群も別物になる。生成コードは再生成で消えるため手を入れられない。

## 決定
- バージョン1つにつきライブラリ1つ(STBridge201/202/210)。
  中身は「生成クラス+埋め込みXSD+ICompare(比較キー)+許容差/重要度設定」
- 比較ロジックの注入はC#の**partial class**で行う
  (生成クラスと同名の型をICompare/フォルダに定義してインターフェースを実装)
- アプリ(デスクトップ/Web)はVersionEngineでバージョンを切り替えて同じライブラリ群を使う

## 理由
- デスクトップとWebで比較ロジックを物理的に共有できる(結果の食い違いが構造的に起きない)
- 新バージョン対応が「ライブラリを1つ足す」定型作業になる(手順はMAINTENANCE.md §5)

## 影響
- csprojのRootNamespaceは旧exe時代の名前(STB_DiffChecker_2XX)を維持している。
  埋め込みXSDのリソース名がこれに依存するため変更しないこと

# 0002: idではなく座標・符号でマッチングする

- ステータス: 採用（元設計の明文化。2026-07）

## 文脈
stbファイルのid/guidは出力ソフトが独自に採番するため、同じ建物でも
解析ソフト出力とBIM出力ではidが一致しない。

## 決定
要素同士の対応付け(どれとどれを比較するか)にidを使わない。
- 部材: 両端(または基準)の**節点座標**。許容差設定の範囲内ならAlmostMatch
- 断面: **符号(name)+階(floor)**
- 配筋・断面形状の配列: **pos / order属性**
- キー定義は各要素型のICompare実装(partial class)が持つ

一方、id参照属性(id_section等)の**値の比較**は、参照先を解決して意味のある値
(符号名・継手符号)に変換してから行う(IProperty実装)。

## 影響
- ICompareのない配列要素は「同型の先頭」と機械的にペアになる。
  複数要素が並びうる型には必ずICompareを用意すること
- 過去にIProperty実装のB側検索にA側のidを使うバグが多発した(2026-07修正)。
  この方式を触るときはA/Bの取り違えに注意
